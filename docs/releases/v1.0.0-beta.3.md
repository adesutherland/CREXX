# CREXX 1.0.0 Beta 3 Release Notes

_Status: work-in-progress planning note for `develop`. Target tag:
`v1.0.0-beta.3`. Do not describe beta 3 as released until that tag and its
release assets exist. Created: 2026-06-17, after the `v1.0.0-beta.2` tag._

CREXX 1.0.0 beta 3 is planned as the Release 1 foundation beta after the beta 2
close-out. It should convert the beta 2 hardening work into the branch state
needed for Release 1 design lock, foundation decisions, package validation, and
final feature completion.

## Timetable

## Planned Scope

Beta 3 should focus on a smaller, more explicit Release 1 contract rather than
new broad language surface:

- Level B lockdown: finish tests and documentation for the stable Level B
  syntax, library surface, UTF/binary rules, references, arrays, collections,
  interfaces/classes, ADDRESS, TRACE, imports, linking, and packaging.
- Level B/G split: publish the Release 1 boundary between stable Level B and
  the initial Level G library overlay.
- UTF and text ownership: keep `.string` as valid UTF-8, `.binary` as
  arbitrary bytes, and move richer Unicode work into Level G library planning.
- Plugin and package policy: classify plugin directories, align default
  build/package contents, and make optional or experimental pieces obvious.
- VM/compiler foundations: harden late load/relink behaviour, indexed lookup,
  large immutable constants, and the measured optimizer work needed by approved
  Release 1 demos.
- Level C milestone: keep parser/highlighter support visible and decide whether
  a small lowering/execution proof belongs in beta 3 or moves after Release 1.
- Packaging: keep portable ZIP assets, continue macOS `.pkg` and Linux `.deb`
  hardening, and add the signed Windows NSIS installer only if the build,
  signing, and upload flow is reliable before the beta 3 tag.
- Demos, tutorials, and performance: curate examples and establish performance
  baselines before claiming optimizer wins.

## Out Of Scope For Beta 3

- A final Release 1 stable contract.
- Full Classic Rexx / Level C runtime compatibility.
- Full Level G Unicode or broad Level L syntax sugar.
- Stable GPU integration or shared-memory VM multithreading.
- Windows MSI, WiX, or `winget` publication.

## Expected Package Shape

The final package list must be checked against the `v1.0.0-beta.3` tag assets.
The intended beta 3 shape is:

- Linux x64: portable ZIP, with `.deb` promoted only if install/uninstall and
  dependency checks are reliable.
- Windows x64: signed portable ZIP, plus a signed NSIS `setup.exe` if the local
  signing flow can build, sign, upload, and verify it reliably.
- macOS arm64 and x86_64: portable ZIPs plus signed, notarized, stapled `.pkg`
  installers when Apple signing and notarization secrets are configured.

## Known Limitations

- Beta 3 remains part of the Release 1 beta line. The Release 1 target remains
  end of August 2026.
- Level B is the main implemented language surface, but final Release 1
  lockdown work is still pending.
- Level G is an initial library overlay, not a full separate language contract.
- Normal Level C compilation is not part of beta 3 unless the final beta 3 tag
  notes explicitly include a small approved lowering proof.
- GPU integration and VM multithreading are not stable beta 3 features.
- Release package signing status, checksums, and final asset names must be
  checked against the actual beta 3 tag assets before publication.
