# crexx Development Workflow

This skill outlines the standard development and debugging workflows for the `crexx` project.

## 1. Building the Project (Debug Mode)

To build the `crexx` toolchain in debug mode:

```bash
cmake --build cmake-build-debug -j 4
```

- If the build fails, analyze the compiler errors/warnings from the terminal output. Do not attempt to fix them unless instructed.

## 2. Running CTests

To run the project's test suite and review the output:

```bash
ctest --test-dir cmake-build-debug --output-on-failure
```

- Review any test failures and their logs, and report the findings back to the user.

## 3. Creating, Compiling, and Running a REXX Program

All modern REXX programs in `crexx` MUST begin with the `OPTIONS LEVELB` instruction.

To compile, assemble, and run a REXX script (e.g., `program.rexx`):

1. **Compile**:
   ```bash
   ./cmake-build-debug/bin/rxc program.rexx
   ```
   *This produces `program.rxas`.*

2. **Assemble**:
   ```bash
   ./cmake-build-debug/bin/rxas program.rxas
   ```
   *This produces `program.rxbin`.*

3. **Run**:
   ```bash
   ./cmake-build-debug/bin/rxvm program.rxbin
   ```

Report the final output to the user.

## 4. Debugging AST Output

To investigate the Abstract Syntax Tree (AST) generated by the Lemon parser for a REXX file:

```bash
./cmake-build-debug/bin/rxc -d2 program.rexx
```

- Capture the terminal output.
- Analyze the dumped AST tree. Identify any malformed branches, missing scope boundaries, or incorrect token types based on the Lemon grammar rules.
